# GraphRAG Audit Knowledge Base

基于知识图谱的审计 RAG（检索增强生成）知识库项目脚手架。

## 项目概述

本项目旨在构建一个面向审计领域的智能问答系统，结合：
- **知识图谱 (Neo4j)**：存储审计法规、案例、组织、风险事件等实体及其关系
- **向量检索 (Chroma)**：存储文档片段的嵌入向量，支持语义相似度检索
- **LLM (LangChain)**：基于检索结果生成符合审计合规要求的回答

## 核心特性

### 1. 混合检索

`HybridRetriever` 同时执行：
- **向量检索**：基于语义相似度的文档片段检索
- **图谱检索**：基于关键词匹配和 Cypher 多跳查询的关系发现
- **结果融合**：加权评分融合策略

### 2. 审计合规约束

RAG 响应强制包含：
- **【依据条款】**：引用的法规条款原文
- **【关联案例】**：相似历史案例
- **【置信度】**：0-1 之间的可信度评分
- **【溯源路径】**：vector/graph 来源标识
- **【校验标志】**：金额/时间二次校验状态
- **【风险等级】**：low/medium/high
- **【合规建议】**：可操作的合规建议

### 3. 图本体设计

核心实体：
- `Organization`：组织机构
- `Regulation`：法规条款
- `AuditCase`：审计案例
- `RiskEvent`：风险事件

核心关系：
- `ISSUED_BY`：法规由...发布
- `APPLIES_TO`：法规适用于
- `INVOLVED_IN`：组织卷入案例
- `VIOLATED`：违反法规
- `DETECTED_IN`：风险事件在...中发现

### 4. 可扩展性

- **抽取模块**：预留 `DocumentLoader` 接入点，支持 PDF/Word/TXT 等多种格式
- **检索模块**：策略模式设计，可替换向量库或图谱查询策略
- **生成模块**：依赖注入 LLM 客户端，支持 OpenAI/本地模型切换

## 注意事项

1. **LLM 配置**：当前项目使用模拟响应，需配置 `.env` 中的 `LLM_API_KEY` 以启用真实 LLM
2. **数据安全**：生产环境请修改默认密码，限制 CORS 域名
3. **性能优化**：大量数据时建议为 Neo4j 增加内存，为 Chroma 配置持久化存储
4. **日志审计**：所有 RAG 查询均有审计日志追踪，可通过 task_id 追溯完整链路
